cli

A module for parsing command line options and arguments, providing both POSIX-compliant parsers and Windows-style parsers. It is very simple in that it merely maps command-line options to callbacks, and

Discussion

It provides the following classes that give you the ability to parse command-line options in different ways:

  • MSDOSCLIParser (For commands of the form "command /s /f:file.txt ...")
  • POSIXCLIParser (For commands of the form "command -sv -f file.txt ...")
  • GNUCLIParser (For commands of the form "command -sv --long-opt -f file.txt ...")


Program Argument Syntax Conventions says the following:

An option and its argument may or may not appear as separate tokens. (In other words, the whitespace separating them is optional.) Thus, ‘-o foo’ and ‘-ofoo’ are equivalent.

But because of the ambiguities that introduces, this library will treat all groups of multiple characters preceded by a single dash '-' as several bundled options, rather than one option followed by an argument with no space between them. In other words will be equivalent to , not .

If an option is not supplied with an argument, the associated callback is executed with the second parameter, , set to an empty string. It is on the developer to handle the absence of an argument within the callback if one is expected for a given option.
